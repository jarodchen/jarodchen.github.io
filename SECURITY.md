# 安全策略

## 报告漏洞

我们非常重视安全问题。如果您发现任何安全漏洞或潜在风险，请按照以下步骤负责任地披露：

### 报告流程

1. **不要**通过公开的 GitHub Issues 报告安全问题
2. 通过 GitHub 的[私有漏洞报告](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing/privately-reporting-a-security-vulnerability)功能私下联系我们

### 期望响应时间

- **初步确认**：48 小时内确认收到报告
- **状态更新**：每 5 个工作日提供修复进展
- **修复目标**：严重漏洞 30 天内修复，其他漏洞 90 天内修复

### 披露政策

- 在漏洞修复之前，请不要公开披露相关信息
- 我们将在修复后与您协调公开披露的时间
- 感谢您帮助我们保护用户和数据的安全

## 支持的安全版本

我们将为以下版本提供安全更新：

| 版本 | 支持状态 |
|------|---------|
| 最新版本 | ✅ 完全支持 |
| 前一个版本 | ✅ 关键安全更新 |
| 更早版本 | ❌ 不再支持 |

## 安全措施

本项目采用以下安全措施：

- ✅ **依赖项扫描**：使用 Dependabot 自动检测依赖项中的安全漏洞
- ✅ **密钥扫描**：防止 API 密钥、令牌等敏感信息泄露
- ✅ **代码扫描**：定期扫描代码中的潜在安全问题
- ✅ **推送保护**：阻止包含机密的代码推送到仓库

## 最佳实践建议

如果您为本项目贡献代码，请遵循以下安全最佳实践：

1. **不要硬编码密钥**：使用环境变量或配置文件管理敏感信息
2. **保持依赖项更新**：定期更新依赖项以获取安全补丁
3. **代码审查**：所有 PR 都需要经过至少一人的代码审查
4. **最小权限原则**：只请求必要的权限

## 联系方式

如有任何安全问题，请通过 GitHub 的私有漏洞报告功能联系我们。
